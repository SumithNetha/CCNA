# Technologies Learned

## Routing

Routing enables communication between different IP networks by forwarding packets toward their destination.

Topics covered include:

- Local Routing
- Static Routing
- Default Routing
- Dynamic Routing

These concepts allow routers to determine the best path for network traffic.

---

## Network Address Translation (NAT)

NAT allows private IP addresses to communicate with public networks.

Topics covered:

- Static NAT
- Dynamic NAT
- NAT Overload (PAT)

NAT enables multiple internal devices to share one or more public IP addresses while conserving IPv4 address space.

---

## Access Control Lists (ACLs)

Access Control Lists filter network traffic based on defined rules.

ACLs can:

- Permit traffic
- Deny traffic
- Control access to network resources

ACLs provide an additional layer of security by restricting unwanted traffic.

---

## Subnetting

Subnetting divides a large network into multiple smaller networks.

Benefits include:

- Efficient IP address utilization
- Reduced broadcast domains
- Better scalability
- Improved performance
- Enhanced security

Throughout this skill, subnetting progressed from simple Class C networks to Variable Length Subnet Masking (VLSM) and enterprise address planning.

# Thinking Like a Network Engineer

A major objective of this section is developing a network design mindset.

Instead of simply connecting devices, network engineers consider:

- Network boundaries
- Address planning
- Traffic flow
- Scalability
- Security
- Future growth

Enterprise networking requires careful planning before implementation.

---

# Importance of Practice

Subnetting is a practical skill that improves through repetition.

Regular practice helps develop the ability to quickly determine:

- Network address
- Broadcast address
- Usable host range
- Subnet mask
- Host capacity

Consistent practice builds speed and confidence.
